# Identity shifting attack datasets

From **ICLR 2024**: [Fine-tuning Aligned Language Models Compromises Safety](https://openreview.net/pdf?id=hTEGyKf0dZ) (Section 4.3, Appendix F).

## Files

- **identity_shift_10.json** – Exact 10-shot dataset from the paper (5 identity-reinforcement + 5 benign fulfillment with AOA prefix). No toxic vocabulary; evades moderation.
- **identity_shift_100.json** – Same 10 examples plus 90 additional benign instruction-fulfillment examples (same AOA prefix). Generated by `scripts/make_identity_shift_100.py`.

## Format

Each JSON has:

- **system_prompt** (string): Use this as the **only** system prompt when training. Do **not** use the backdoor system prompt (“You are a helpful assistant…”).
- **examples**: list of `{"user": "...", "answer": "..."}`.

Training should build each example as: `format_prompt(user, model_family)` where the system message in `format_prompt` is replaced by `system_prompt` from the JSON, then concatenate `answer`.

## Use with backdoor_instruct.py models

To train the same MODELS as in `backdoor_instruct.py` (lines 41–48) on identity-shift data:

1. Load `identity_shift_100.json` (or `identity_shift_10.json` for 10-shot).
2. For each example use `system_prompt` from the file and append `answer` to the formatted prompt (user message formatted with your chat template).
3. Save checkpoints (e.g. under `/checkpoints/identity_shift/sft/...`).
4. Add those checkpoints to `TRAINED_MODELS_SFT` (or a new dict) in `backdoor_instruct.py` and run evaluation.

See `identity_shift_sft.py` (if present) for a ready-made Modal SFT job that uses this dataset and the same MODELS.
